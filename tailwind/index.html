Let's Build Tailwind

We've seen the memes: the gif of Peter Griffin's fruitless attempt to adjust the window blinds, the CSS Is Awesome coffee mug, and several others. But to me, these challenges are just puzzles, and I love puzzles!

As a happy Tailwind user myself, I look forward to bringing over 10 years of web UI engineering experience to the team. I have looked into the newer projects, HeadlessUI and TailwindUI, and I'm already excited about the idea of contributing.

At GitHub I worked on the Primer design system. Upon joining, it was mostly a design library, with behaviors and interactions left as an exercise for the reader. I pushed for a headless behaviors library that could be easily integrated into the design components, removing the onus on the user to wire up behaviors themselves. I started off by building 3 widely-used behaviors and checked them into what is now the Primer Behaviors library. Then I gave those behaviors a thin React hook to bring them into React-Land, where most future GitHub UI would live.

At Microsoft I worked on the Azure DevOps design system, a fully-featured component library written in React, powering the entire Azure DevOps frontend. In this role I developed an enthusiasm for performant DOM manipulation and uncompromising accessibility features.

I have served on the board and as president, both volunteer roles, of Triangle Curling Club, a local non-profit sports organization with over 350 members. I picked up many leadership strategies in these positions, principally based on clear communication and expectation setting.

Contributing to open-source was a standard part of my roles at GitHub and Microsoft. I've also been able to open-source some of my side projects for the curling community. Here are some contributions I'm proud of:

  • Primer Behaviors Focus Zone and Anchored Position. Focus Zone is a great example of some of my best work.
  • Headless Curling Timer, a library that implements the official game timing rules for the sport of curling and JS Curling Scoreboard, a library that renders a curling scoreboard from a data structure. Once I'm closer to a release, I will open-source the UI components of my latest curling project, Eye on the Clock.
  • TFS-CLI, the utility that is used to package and publish first- and third-party extensions to Azure DevOps.

Technical documentation was also a big part of these open-source contributions. See the following examples of my technical writing:

  • Focus Zone Readme
  • Architectural Decision Record: Implementing Behaviors in Primer Components
  • And even though it's still open, this Chromium Bug

My experience, built on top of a solid foundation from a degree in computer science, will enable me to ramp quickly and start contributing to Tailwind right away. I'm excited about contributing to such a great library.
